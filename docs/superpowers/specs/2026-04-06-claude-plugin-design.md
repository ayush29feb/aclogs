# AC Logs Claude Plugin Design

## Goal

Convert the AC Logs repo into an installable Claude Code plugin so anyone can run `claude plugin install ayush29feb/aclogs` and get the full app — server, dashboard, CLI, and Claude skills — in one command.

## Architecture

The existing app code (server/, dashboard/, cli/, workouts/) is unchanged. A thin plugin infrastructure layer is added on top: skills, hooks, and plugin metadata. The database moves out of the repo into `~/.aclogs/data/` so it persists across plugin updates.

## Plugin structure

```
aclogs/
├── skills/
│   ├── setup.md              # first-time + re-setup flow
│   ├── start-servers.md      # start GraphQL server + dashboard
│   ├── stop-servers.md       # kill both servers
│   ├── import-workouts.md    # run CLI import script
│   └── remote-control.md     # Tailscale setup + phone URL
├── hooks/
│   └── hooks.json            # registers plugin, no auto-start
├── CLAUDE.md                 # lightweight context: "this is AC Logs, use the skills"
├── package.json              # plugin metadata: name, version
├── setup.sh                  # updated to write data to ~/.aclogs/data/
├── server/
├── dashboard/
├── cli/
└── workouts/
```

## Data persistence

- Database: `~/.aclogs/data/gym.db` (outside plugin directory, survives updates)
- `server/.env`: generated by setup.sh pointing to `~/.aclogs/data/gym.db` (absolute path)
- Workouts markdown files: stay in `workouts/` inside the plugin (source files, not user data)
- `setup.sh` creates `~/.aclogs/data/` if it doesn't exist

## Skills

### setup.md
Mirrors the existing CLAUDE.md setup flow (Steps 1–5). Handles first-time and re-setup. Asks permission before running commands. Runs `setup.sh` from the plugin root.

### start-servers.md
Kills any existing processes on ports 47322/47323, then starts the GraphQL server and dashboard. Checks `lsof` to confirm both are listening. Reports URLs.

### stop-servers.md
Kills processes on ports 47322/47323.

### import-workouts.md
Runs `uv run python import_workouts.py` from `cli/`. Warns that this wipes and re-imports all data.

### remote-control.md
Checks Tailscale is running, gets the device IP via `tailscale ip -4`, gives the user the dashboard URL to open on their phone.

## Install flow

```
claude plugin install ayush29feb/aclogs
```
1. Claude Code clones repo into `~/.claude/plugins/cache/ayush29feb/aclogs/`
2. Post-install runs `setup.sh` — installs Node deps, builds server, generates Prisma client, generates Relay types, creates `~/.aclogs/data/`, writes `server/.env`
3. Done. From any Claude session the user can say "set up AC Logs", "start AC Logs servers", etc. and Claude uses the skills.

## Servers

No auto-start on session open — not all Claude sessions are AC Logs related. User asks Claude explicitly. Skills handle start/stop/status.

App code runs from the plugin directory (`~/.claude/plugins/cache/ayush29feb/aclogs/server/`, etc.) — same pattern as superpowers' visual companion server.

}

## What doesn't change

GraphQL server, React dashboard, Python CLI, Prisma schema, import script, workouts markdown files — all untouched. The plugin layer is purely additive.
